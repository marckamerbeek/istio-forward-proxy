 Configure HTTP_PROXY in your pods
============================================================

Deploy pods in a namespace with the istio.io/dataplane-mode=ambient label
and set the HTTP_PROXY environment variable:

    apiVersion: v1
    kind: Pod
    metadata:
      name: my-app
      namespace: team-a
    spec:
      containers:
        - name: app
          image: ...
          env:
            - name: HTTP_PROXY
              value: "http://{{ include "forward-proxy.fullname" . }}.{{ .Release.Namespace }}.svc.cluster.local:{{ $port }}"
            - name: HTTPS_PROXY
              value: "http://{{ include "forward-proxy.fullname" . }}.{{ .Release.Namespace }}.svc.cluster.local:{{ $port }}"

============================================================
 Register external hosts
============================================================

Create a ServiceEntry (in any namespace) to allow traffic to an external host:

    apiVersion: networking.istio.io/v1
    kind: ServiceEntry
    metadata:
      name: example-com
      namespace: team-a
    spec:
      hosts:
        - example.com
      ports:
        - number: 443
          name: https
          protocol: HTTPS
        - number: 80
          name: http
          protocol: HTTP
      resolution: DNS
